This guide walks you through registering an account, obtaining a JWT token, and making your first authenticated request against the USM Fitness API.

<Note>
  You need a running instance of the USM Fitness backend. All examples use `http://localhost:8000` as the base URL. Replace this with your deployed server URL if needed.
</Note>

<Steps>
  <Step title="Register a new account">
    Create a user account by posting your RUT (used as the username), password, and academic details to `/api/usuarios/registro/`.

    <CodeGroup>

    ```bash curl
    curl --request POST \
      --url http://localhost:8000/api/usuarios/registro/ \
      --header 'Content-Type: application/json' \
      --data '{
        "username": "20.123.456-7",
        "password": "MySecurePass123",
        "first_name": "Ana",
        "last_name": "González",
        "carrera": "Ingeniería Civil"
      }'
    ```

    ```python Python
    import requests

    response = requests.post(
        "http://localhost:8000/api/usuarios/registro/",
        json={
            "username": "20.123.456-7",
            "password": "MySecurePass123",
            "first_name": "Ana",
            "last_name": "González",
            "carrera": "Ingeniería Civil",
        },
    )
    print(response.json())
    ```

    </CodeGroup>

    A successful registration returns:

    ```json
    {
      "mensaje": "Usuario registrado exitosamente."
    }
    ```

    <Tip>
      The `username` field must be your RUT in Chilean format, for example `20.123.456-7`. This is the same value you will use to log in.
    </Tip>
  </Step>

  <Step title="Log in to get tokens">
    Exchange your RUT and password for an access token and a refresh token.

    <CodeGroup>

    ```bash curl
    curl --request POST \
      --url http://localhost:8000/api/login/ \
      --header 'Content-Type: application/json' \
      --data '{
        "username": "20.123.456-7",
        "password": "MySecurePass123"
      }'
    ```

    ```python Python
    import requests

    response = requests.post(
        "http://localhost:8000/api/login/",
        json={
            "username": "20.123.456-7",
            "password": "MySecurePass123",
        },
    )
    tokens = response.json()
    access_token = tokens["access"]
    print(access_token)
    ```

    </CodeGroup>

    The response contains two tokens:

    ```json
    {
      "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
    ```

    The `access` token is valid for **1 day**. The `refresh` token is valid for **7 days**. See [Authentication](/authentication) for details on refreshing expired tokens.
  </Step>

  <Step title="Make an authenticated request">
    Pass the access token in the `Authorization` header as a Bearer token to fetch your own profile.

    <CodeGroup>

    ```bash curl
    curl --request GET \
      --url http://localhost:8000/api/usuarios/me/ \
      --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
    ```

    ```python Python
    import requests

    access_token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

    response = requests.get(
        "http://localhost:8000/api/usuarios/me/",
        headers={"Authorization": f"Bearer {access_token}"},
    )
    print(response.json())
    ```

    </CodeGroup>

    A successful response returns your user profile including your role level and career.
  </Step>

  <Step title="Check available gym blocks">
    Retrieve the list of upcoming gym time blocks that students can reserve.

    <CodeGroup>

    ```bash curl
    curl --request GET \
      --url http://localhost:8000/api/bloques/ \
      --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
    ```

    ```python Python
    import requests

    access_token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

    response = requests.get(
        "http://localhost:8000/api/bloques/",
        headers={"Authorization": f"Bearer {access_token}"},
    )
    blocks = response.json()
    for block in blocks:
        print(block)
    ```

    </CodeGroup>

    The response is a list of available time blocks, each showing capacity and current reservation count.

    <Note>
      The weekly schedule is auto-generated every Monday. If no blocks appear, the schedule may not have been generated yet for the current week.
    </Note>
  </Step>
</Steps>
