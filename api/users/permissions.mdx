---
title: 'Update permissions'
description: "Update a user's role, sports selection status, and sport branch. Admin only."
---

Partially update a user's permissions, including their role, whether they are part of a sports selection team, and which sport branch they belong to.

<Warning>
  This endpoint requires Admin role (`rol == 3`). Requests from non-admin users return 403.
</Warning>

## Endpoint

```
PATCH /api/usuarios/{id}/actualizar_permisos/
```

## Request

### Path parameters

<ParamField path="id" type="integer" required>
  The internal database ID of the user whose permissions are being updated.
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  `Bearer <access_token>` — must belong to a user with `rol == 3`.
</ParamField>

### Body parameters

<ParamField body="rol" type="integer">
  The new role to assign. Accepted values:

  - `1` — Student
  - `2` — Trainer
  - `3` — Admin
</ParamField>

<ParamField body="es_seleccionado" type="boolean">
  Whether the user is part of a USM sports selection team.
</ParamField>

<ParamField body="rama_deportiva" type="string">
  The sport branch to assign to the user (e.g., `"Fútbol"`, `"Natación"`).
</ParamField>

<Note>
  Setting `es_seleccionado` to `false` automatically clears the `rama_deportiva` field, regardless of the value submitted. You do not need to explicitly clear it.
</Note>

## Response

### 200 — Success

<ResponseField name="mensaje" type="string">
  Confirmation message indicating the permissions were updated.
</ResponseField>

### 403 — Forbidden

Returned when the requesting user does not have Admin role.

```json
{
  "error": "No tienes permisos de Administrador."
}
```

## Example

<RequestExample>

```bash cURL
curl --request PATCH \
  --url https://your-domain.com/api/usuarios/42/actualizar_permisos/ \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...' \
  --header 'Content-Type: application/json' \
  --data '{
    "rol": 2,
    "es_seleccionado": true,
    "rama_deportiva": "Natación"
  }'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "mensaje": "Permisos del usuario actualizados correctamente."
}
```

```json 403
{
  "error": "No tienes permisos de Administrador."
}
```

</ResponseExample>

## Business rules

- Sending `es_seleccionado: false` clears `rama_deportiva` automatically on the server side. The field is set to an empty value regardless of what was submitted alongside it.
- All three fields (`rol`, `es_seleccionado`, `rama_deportiva`) are optional in a single PATCH request. Only the fields included in the request body are updated.
